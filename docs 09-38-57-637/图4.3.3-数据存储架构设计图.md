## 图 4.3.3 数据存储架构设计图

**对应小节**：4.3.3 数据存储架构设计  
**图注建议**：系统采用“运行时状态 + 本地持久化”双层数据机制，核心对象包括 Resume、Setting 与 AI任务状态，实现可恢复与一致性保障。

```mermaid
%%{init: {
  "theme": "base",
  "themeVariables": {
    "background": "#ffffff",
    "mainBkg": "#ffffff",
    "secondaryColor": "#ffffff",
    "tertiaryColor": "#ffffff",
    "primaryColor": "#ffffff",
    "primaryTextColor": "#000000",
    "primaryBorderColor": "#333333",
    "lineColor": "#333333",
    "clusterBkg": "#ffffff",
    "clusterBorder": "#cccccc",
    "titleColor": "#000000"
  },
  "flowchart": {
    "curve": "linear",
    "padding": 12,
    "nodeSpacing": 50,
    "rankSpacing": 50
  }
}}%%
flowchart TB
    FE[前端业务模块]
    STORE[运行时状态层\nZustand Stores]

    subgraph DB[IndexedDB 持久化层]
        direction LR
        R[(resumes_data\nResume对象)]
        S[(settings\nSetting对象)]
        T[(task_cache\n任务阶段结果/日志)]
    end

    FE --> STORE
    STORE --> R & S & T

    R --> LOAD[页面加载恢复]
    S --> CFG[模型与接口配置恢复]
    T --> LOG[任务进度与结果回显]

    LOAD --> FE
    CFG --> FE
    LOG --> FE

    classDef plain fill:#ffffff,stroke:#333333,color:#000000
    class FE,STORE,R,S,T,LOAD,CFG,LOG plain
    style DB fill:#ffffff,stroke:#cccccc
```

---

## 使用说明

1. 打开 [Mermaid Live Editor](https://mermaid.live)。
2. 复制上方代码块（从 `%%{init` 到 `style DB` 行）。
3. 连线为折线/直线段（`curve: linear`），画布与子图为白底；导出 PNG 若背景非纯白，可用 SVG 后铺 `#ffffff`。
4. 若旧版 Mermaid 不支持 `STORE --> R & S & T`，可拆成三行分别连接。
5. 点击 **Actions → PNG** 或 **SVG** 导出图片。
6. 插入论文并标注图号为「图 4.3.3 数据存储架构设计图」。
