## 图 4.3.1 系统总体架构设计图

**对应小节**：4.3.1 系统总体架构设计  
**图注建议**：系统采用四层总体架构：表示层、业务层、AI能力层和数据层，各层职责清晰并通过接口协同。

```mermaid
%%{init: {
  "theme": "base",
  "themeVariables": {
    "background": "#ffffff",
    "mainBkg": "#ffffff",
    "secondaryColor": "#ffffff",
    "tertiaryColor": "#ffffff",
    "primaryColor": "#ffffff",
    "primaryTextColor": "#000000",
    "primaryBorderColor": "#333333",
    "lineColor": "#333333",
    "clusterBkg": "#ffffff",
    "clusterBorder": "#cccccc",
    "titleColor": "#000000"
  },
  "flowchart": {
    "curve": "linear",
    "padding": 12,
    "nodeSpacing": 50,
    "rankSpacing": 50
  }
}}%%
flowchart TB
    U[求职用户]

    subgraph L1[表示层]
        direction LR
        A[Landing / Dashboard]
        B[简历编辑器]
        C[AI Lab]
        D[设置中心]
    end

    subgraph L2[业务层]
        direction LR
        E[业务服务层\nHooks / Services]
        F[状态管理层\nZustand]
        G[路由适配层\nAPI Routes]
    end

    subgraph L3[AI能力层]
        direction LR
        H[Chain编排]
        I[Graph工作流\nResearch/Analyze/Rewrite]
        J[多模型适配]
    end

    subgraph L4[数据层]
        direction LR
        K[IndexedDB\n简历/配置]
        M[任务状态数据\n日志/中间结果]
    end

    U --> A & B & C & D

    A --> E
    B --> E
    C --> E
    D --> E

    E --> F
    E --> G
    G --> H --> I --> J

    F --> K
    E --> M

    classDef plain fill:#ffffff,stroke:#333333,color:#000000
    class U,A,B,C,D,E,F,G,H,I,J,K,M plain
    style L1 fill:#ffffff,stroke:#cccccc
    style L2 fill:#ffffff,stroke:#cccccc
    style L3 fill:#ffffff,stroke:#cccccc
    style L4 fill:#ffffff,stroke:#cccccc
```

---

## 使用说明

1. 打开 [Mermaid Live Editor](https://mermaid.live)。
2. 复制上方代码块（从 `%%{init` 到 `style L4` 行）。
3. 连线为折线/直线段（`curve: linear`），画布与子图为白底；导出 PNG 若背景非纯白，可用 SVG 后铺 `#ffffff`。
4. 若旧版 Mermaid 不支持 `U --> A & B & C & D`，可拆成四行分别连接。
5. 点击 **Actions → PNG** 或 **SVG** 导出图片。
6. 插入论文并标注图号为「图 4.3.1 系统总体架构设计图」。
